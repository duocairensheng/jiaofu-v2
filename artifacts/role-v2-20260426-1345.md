# 角色权限系统 V2 — 客服角色 + 客户自动同步 + 报单分析权限强化

## 目标
1. 客户管理同步报单中的客户名称，报单人自动默认为该客户的负责人
2. 新增"客服"角色：权限=区总，但不可修改系统参数
3. 保留区总为推广经理分配/调整客户的功能
4. 推广经理在仪表盘和数据分析中只显示所属客户的数据

## 改动文件
- `app.js`：80,372 bytes（11 处 patch 全部通过）
- `index.html` / `styles.css`：无改动

## 核心改动明细

### 1. 新增 `isPromotionManager()` 函数
```js
function isPromotionManager() { var u = getCurrentUser(); return u && u.role === '推广经理'; }
```
用于数据过滤（区别于 `isRegionalManager` 仅用于系统设置权限）

### 2. `initUsers()` — 新增客服账号
| 用户名 | 密码 | 角色 | 显示名 |
|--------|------|------|--------|
| admin | admin123 | 区总 | 管理员 |
| pm001 | pm001123 | 推广经理 | 张经理 |
| pm002 | pm002123 | 推广经理 | 李经理 |
| **kefu** | **kefu123** | **客服** | **客服专员** |

### 3. 权限矩阵（3 角色）
| 功能 | 区总 | 客服 | 推广经理 |
|------|:--:|:--:|:--:|
| 查看所有客户 | ✓ | ✓ | ✗ (仅分配) |
| 分配/删除客户 | ✓ | ✓ | ✗ |
| 查看所有报单 | ✓ | ✓ | ✗ (仅分配) |
| 仪表盘全量统计 | ✓ | ✓ | ✗ (仅分配) |
| 数据分析全量 | ✓ | ✓ | ✗ (仅分配) |
| 系统设置 | ✓ | ✗ | ✗ |
| 侧边栏隐藏设置 | ✗ | ✓ | ✓ |

### 4. 报单保存 → 自动同步客户
新增报单时，如果客户名称不在客户列表中，自动创建客户记录，`assignedTo` 设为当前用户（报单人）。

### 5. Bug 修复
- `renderOrdersTable` 中角色过滤代码原来放在 `var filtered = data;` 之前，导致 `filtered` 未定义就被引用。已移至正确位置。

## 权限判断逻辑

```js
// 仅区总可改系统参数
isRegionalManager()    → role === '区总'

                              → role === '推广经理'

: 看所有数据（区总 or 客服）
看数据（非推广经理）
```

（用于按钮、客户列表等：显示完整权限按钮 / 返回全量数据）

数据过滤（报单/仪表盘/分析：仅推广经理的数据被限制）
